# Entropy 0.1.0 Beta 1

Entropy is a modern app for programmable keyboards and input devices.

This is the first public beta. It is intended for users who are comfortable testing
new desktop tooling with Vial-compatible devices and reporting firmware/device
specific issues.

Internal reference: `v1.13.136`.

## Downloads

The GitHub release package should contain:

- `entropy-0.1.0-beta.1-linux-x86_64.AppImage`
- `entropy-0.1.0-beta.1-windows-x86_64.zip`
- `SHA256SUMS.txt`

Windows builds are unsigned during beta, so Windows SmartScreen may show a warning.

## What Is Included

- Visual layout editor for Vial-compatible devices.
- Layer editing and key assignment.
- Keycode picker with Basic, Symbols, Modifiers, Special, RGB, Macro, Tap Dance, and Custom sections.
- Combo, Macro, Tap Dance, and Key Override editors.
- Advanced firmware settings for supported QMK/Vial features.
- RGB and Layer LED controls where supported by firmware.
- Matrix Tester for supported Vial devices.
- Layout Indicator companion window.
- Text Expander and Universal Symbols desktop integrations.
- Linux udev rule installer for Vial device access.
- Linux IBus and Fcitx5 helper backends for Wayland input workflows.

## Compatibility

Entropy currently targets Vial-compatible HID devices. The interface is designed
for keyboards and adjacent programmable input devices such as macropads, trackballs,
touchpads, encoders, and display modules when firmware exposes those capabilities.

Best-tested devices are Ergohaven devices and Vial-compatible QMK/RMK-style builds.

## Known Limitations

- This is a beta release; device-specific firmware differences may still expose bugs.
- Windows builds are portable and unsigned.
- Linux users may need to install Vial udev rules before Entropy can open devices.
- macOS packaging exists in the repository, but the first beta package focuses on Linux and Windows.
- ZMK Studio is not supported.
- Browser-only and mobile workflows are not supported.

## Reporting Issues

Please include:

- Operating system and version.
- Entropy public release version and internal build reference.
- Device model and firmware source/version when known.
- Whether the device is locked or unlocked in Vial.
- Steps to reproduce.
- Screenshots or logs when useful.
